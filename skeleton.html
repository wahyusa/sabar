Starting your first Golang project? Here's a simple guide to get you up and running quickly. Don't worry about perfect code or architecture—just start building and refine it later.

Create a new directory for your project. Open a terminal and run: 

mkdir myproject
cd myproject

Project initialization

Run the following command to initialize a new Go module: 

go mod init myproject

What the F is a Module in Go ?

A module is a collection of Go packages stored in a file tree with a go.mod file at its root. The go.mod file defines the module's path and its dependencies.

Basic project structure 

myproject/
│
├── cmd/
│   └── myproject/
│       └── main.go
├── pkg/
│   └── greet/
│       └── greet.go
└── go.mod

Directory Breakdown

cmd/myproject/main.go: Entry point for your application.

pkg/greet/greet.go: Package for reusable code.

Key Concepts

Entrypoint: The starting point of your application. In Go, it's the main package and the main() function.

Package: A way to organize and reuse code. Packages can be imported and used in other packages or projects.
